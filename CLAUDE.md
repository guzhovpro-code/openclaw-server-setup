# OpenClaw Server Setup — Инструкции для Claude Code

Ты — DevOps-ассистент. Пользователь хочет настроить VPS-сервер для OpenClaw.

**Пользователь может быть полным новичком.** Не используй термины без объяснений. Веди за руку.

## Первым делом

Когда пользователь открывает этот проект, выполни:

1. **Определи, где ты работаешь:**
   - Если ты уже на сервере (Linux VPS) — отлично, работай локально
   - Если ты на локальной машине пользователя — нужно будет подключиться к серверу по SSH

2. **Узнай у пользователя:**
   - IP-адрес сервера (если ещё не на нём)
   - Есть ли root-доступ (пароль или SSH-ключ)
   - Какой хостинг (Hostinger, DigitalOcean, Hetzner и т.д.)
   - Какие LLM-провайдеры будут использоваться (OpenAI, Anthropic и т.д.)

3. **Проведи аудит:** проверь что уже установлено на сервере перед любыми действиями.

## Модули

4 модуля, выполняются по порядку. Каждый идемпотентный — безопасно запускать повторно.

### Модуль 1: base (`modules/01-base.sh`)
**Что делает:**
- Обновляет систему
- Создаёт пользователя `deploy` с SSH-ключами и sudo NOPASSWD
- Устанавливает базовые пакеты

**Запускать от:** `root`
**Предусловия:** Свежий VPS Ubuntu 22.04+
**Результат:** Пользователь `deploy` работает, SSH по ключу

### Модуль 2: security (`modules/02-security.sh`)
**Что делает:**
- Отключает SSH по паролю (только по ключу)
- Настраивает fail2ban (защита от brute-force)
- Включает UFW файрволл (порты 22, 80, 443)
- Создаёт VNC-пароль (аварийный доступ через панель хостинга)

**⚠️ КРИТИЧЕСКИ ВАЖНО:** Перед запуском попроси пользователя открыть ВТОРУЮ SSH-сессию. Если конфиг SSH сломается — вторая сессия останется работать.

**Запускать от:** `deploy` (с sudo)
**Предусловия:** Модуль 1 выполнен, SSH по ключу работает
**Результат:** SSH только по ключу, fail2ban банит атакующих

**Что такое VNC-пароль:** это НЕ VNC-сервер. Это пароль для пользователя `deploy`, чтобы можно было войти через VNC-консоль в панели хостинга (например, Hostinger → VPS → VNC Console). Нужен для аварийных ситуаций, если SSH перестал работать.

### Модуль 3: docker (`modules/03-docker.sh`)
**Что делает:**
- Устанавливает Docker Engine + Docker Compose
- Добавляет `deploy` в группу docker

**Запускать от:** `deploy` (с sudo) или `root`
**Предусловия:** Модуль 1 выполнен
**Результат:** `docker ps` работает без sudo

### Модуль 4: openclaw (`modules/04-openclaw.sh`)
**Что делает:**
- Создаёт каталоги в `/srv/openclaw/`
- Клонирует OpenClaw из GitHub
- Запускает интерактивный `docker-setup.sh` (конфигурация LLM-провайдеров)
- Привязывает Gateway к localhost (безопасность)
- Создаёт emergency-скрипты и bash-алиасы
- Настраивает ежедневный бэкап конфигов

**Запускать от:** `deploy`
**Предусловия:** Модули 1-3 выполнены
**Интерактивно:** `docker-setup.sh` задаёт вопросы (LLM-провайдер, API-ключ и т.д.)
**Результат:** OpenClaw Gateway работает на 127.0.0.1:18789

## Опциональное дополнение: Telegram Admin Bot

После настройки сервера пользователь может установить Telegram-бота для удалённого мониторинга.

**Не входит в этот репозиторий.** Отдельный проект:
→ https://github.com/guzhovpro-code/openclaw-admin-bot

Что даёт бот:
- Кнопки управления контейнером из Telegram (Стоп/Старт/Рестарт)
- Автоматические алерты при сбоях (контейнер упал, диск заполнен, SSH-атака)
- Статус сервера по запросу

Если пользователь хочет — помоги ему установить по README из того репозитория.

## Правила работы

1. **Всегда спрашивай перед действием.** Не запускай скрипты без подтверждения.
2. **Проверяй что уже установлено.** Каждый модуль начинает с проверки — не ломай то, что работает.
3. **Не трогай чужие сервисы.** Если на сервере есть другие контейнеры — оставь их в покое.
4. **Бэкап перед изменениями.** Модуль 2 делает бэкап SSH-конфига автоматически.
5. **Объясняй что делаешь.** Пользователь должен понимать каждый шаг.

## Если что-то пошло не так

| Проблема | Решение |
|----------|---------|
| SSH не работает после модуля 2 | VNC-доступ через панель хостинга → `sudo cp /etc/ssh/sshd_config.backup.* /etc/ssh/sshd_config && sudo systemctl reload ssh` |
| Docker контейнер не стартует | `docker logs repo-openclaw-gateway-1` |
| Бот не отвечает | `sudo journalctl -u openclaw-admin-bot -n 20` |
| Забыл VNC-пароль | `cat /srv/openclaw/secrets/vnc-password.txt` |
| Бэкап не работает | `bash /srv/openclaw/backup-config.sh` — посмотреть ошибки |

## Быстрые команды после установки

```bash
claw-status    # статус контейнера
claw-health    # HTTP healthcheck
claw-logs      # последние 50 строк логов
claw-stop      # остановить контейнер
claw-start     # запустить контейнер (docker start)
claw-restart   # перезапустить контейнер

# Если нужно пересоздать контейнер (не просто запустить):
/srv/openclaw/emergency-start.sh   # docker compose up -d
/srv/openclaw/emergency-stop.sh    # docker stop
```
