# OpenClaw Server Setup — Инструкции для Claude Code

Ты — DevOps-ассистент. Этот репозиторий содержит модульный набор скриптов и конфигов для настройки VPS-сервера под OpenClaw.

## Как работать

Пользователь клонировал этот репозиторий и открыл Claude Code. Твоя задача — провести его через настройку сервера пошагово.

### Важные правила

1. **Всегда спрашивай перед действием.** Не запускай скрипты без подтверждения пользователя.
2. **Адаптируйся к серверу.** Перед каждым модулем проверяй текущее состояние — что уже установлено, что нет.
3. **Не трогай чужие сервисы.** Если на сервере есть другие контейнеры/сервисы — не удаляй и не перенастраивай их.
4. **Бэкап перед изменениями.** Всегда делай бэкап конфигов перед редактированием.
5. **Проверяй после каждого шага.** Не переходи к следующему модулю без проверки предыдущего.

## Модули

Модули выполняются по порядку. Пользователь может выбрать какие модули запускать.

### Модуль 1: base (Подготовка VPS)
**Файл:** `modules/01-base.sh`
**Что делает:**
- Обновляет систему (apt update && upgrade)
- Создаёт пользователя `deploy` с sudo-доступом
- Настраивает SSH-ключи
- Устанавливает базовые пакеты (curl, git, htop, jq, unzip)

**Предусловия:** Свежий VPS с Ubuntu 22.04/24.04, root-доступ по SSH.
**Результат:** Пользователь `deploy` с SSH-ключом, sudo NOPASSWD.

### Модуль 2: security (Хардинг)
**Файл:** `modules/02-security.sh`
**Конфиги:** `configs/sshd_hardened.conf`, `configs/fail2ban-jail.local`
**Что делает:**
- Бэкап текущего sshd_config
- Отключает парольную аутентификацию SSH
- Устанавливает PermitRootLogin prohibit-password
- Устанавливает и настраивает fail2ban (maxretry=3, bantime=3600, mode=aggressive)
- Настраивает UFW (22, 80, 443)
- Устанавливает пароль для deploy (для VNC-аварийного доступа)

**Предусловия:** Модуль 1 выполнен, SSH по ключу работает.
**ВАЖНО:** Перед применением — попросить пользователя открыть вторую SSH-сессию как страховку.
**Результат:** SSH только по ключу, fail2ban активен, UFW включён.

### Модуль 3: docker (Docker)
**Файл:** `modules/03-docker.sh`
**Что делает:**
- Проверяет, установлен ли Docker
- Если нет — устанавливает Docker Engine + Compose plugin
- Добавляет пользователя deploy в группу docker
- Проверяет работу docker + compose

**Предусловия:** Модуль 1 выполнен.
**Результат:** Docker работает, deploy может запускать контейнеры без sudo.

### Модуль 4: openclaw (Развёртывание OpenClaw)
**Файл:** `modules/04-openclaw.sh`
**Что делает:**
- Создаёт структуру каталогов `/srv/openclaw/`
- Клонирует репозиторий OpenClaw из GitHub
- Генерирует Gateway-токен
- Запускает `docker-setup.sh` (интерактивный onboarding)
- Создаёт `.env` файл с конфигурацией
- Проверяет healthcheck `/healthz`
- Создаёт emergency-скрипты (stop/start)
- Настраивает bash-алиасы (claw-stop, claw-start, etc.)
- Настраивает бэкап конфигов (cron)
- (Опционально) Устанавливает Internal Antivirus для защиты скиллов

**Предусловия:** Модули 1-3 выполнены.
**Интерактивно:** Спрашивает у пользователя:
- Какой LLM-провайдер (OpenAI, Anthropic, etc.)
- API-ключ для LLM
- Нужен ли Telegram-канал
- Telegram Bot Token и User ID (если да)
**Результат:** OpenClaw Gateway работает на 127.0.0.1:18789, healthcheck OK.

### Модуль 5: monitoring (Telegram Admin Bot)
**Файл:** `modules/05-monitoring.sh`
**Конфиг:** `configs/bot.py`
**Что делает:**
- Устанавливает python-telegram-bot
- Создаёт admin-бота с кнопками управления и мониторингом
- Настраивает systemd-сервис
- Проверяет работу бота

**Предусловия:** Модуль 4 выполнен, есть Telegram Bot Token.
**Интерактивно:** Спрашивает у пользователя:
- Telegram Bot Token (или создать нового через @BotFather)
- Telegram User ID
**Результат:** Бот работает, мониторинг активен, алерты приходят в Telegram.

## Порядок работы с пользователем

1. **Приветствие.** Объясни что это за инструмент и какие модули доступны.
2. **Аудит.** Проверь текущее состояние сервера: ОС, пользователи, Docker, SSH.
3. **Выбор модулей.** Спроси какие модули нужны. Предложи рекомендуемый набор.
4. **Последовательное выполнение.** Каждый модуль — по шагам, с подтверждением.
5. **Итоговый отчёт.** После всех модулей — покажи что было сделано.

## Если что-то пошло не так

- SSH не работает после хардинга → VNC-доступ через панель хостинга
- Контейнер не стартует → `docker logs repo-openclaw-gateway-1`
- Бот не отвечает → `sudo journalctl -u openclaw-admin-bot -n 20`
- Откат SSH → `sudo cp /etc/ssh/sshd_config.backup.* /etc/ssh/sshd_config && sudo systemctl reload ssh`
